# Руководство пользователя Canbas

## 1. Назначение системы

Canbas — это веб-приложение для командного управления задачами. Система предназначена для создания команд, организации проектов, постановки задач, назначения исполнителей, отслеживания статусов, хранения комментариев и вложений, а также просмотра аналитической информации.

Руководство предназначено для пользователей, которые впервые начинают работать с системой.

## 2. Требования для запуска

Для начала работы необходимо:

1. установить Python и зависимости проекта;
2. запустить приложение;
3. открыть систему в браузере;
4. перейти на главную страницу сервиса.

Если приложение уже развернуто, пользователю достаточно открыть ссылку в браузере и перейти к регистрации или авторизации.

## 3. Регистрация в системе

Чтобы создать учётную запись, выполните следующие действия:

1. откройте главную страницу приложения;
2. нажмите кнопку **«Начать бесплатно»** или перейдите на страницу регистрации;
3. введите имя пользователя;
4. укажите адрес электронной почты;
5. введите пароль и подтверждение пароля;
6. нажмите кнопку **«Зарегистрироваться»**.

После успешной регистрации система автоматически выполнит вход и перенаправит пользователя в панель управления.

## 4. Вход в систему

Если учётная запись уже существует, выполните вход:

1. откройте страницу авторизации;
2. введите адрес электронной почты;
3. введите пароль;
4. нажмите кнопку **«Войти»**.

После успешной авторизации откроется панель управления пользователя.

[СКРИНШОТ: Страница входа в систему. Скринить форму авторизации.]

## 5. Работа с панелью управления

Панель управления является основной страницей после входа в систему. На ней отображаются:

- список команд пользователя;
- список проектов;
- задачи, назначенные текущему пользователю;
- кнопки быстрого создания проекта и команды;
- краткая сводная информация по числу команд, проектов и задач.

Чтобы начать работу:

1. войдите в систему;
2. ознакомьтесь с карточками статистики в верхней части панели;
3. используйте блоки **«Команды»**, **«Проекты»** и **«Назначено вам»** для перехода к нужному разделу.

[СКРИНШОТ: Панель управления пользователя. Скринить dashboard после входа в систему.]

## 6. Создание команды

Для создания новой команды выполните следующие действия:

1. на панели управления нажмите кнопку создания команды;
2. в открывшейся форме укажите название команды;
3. нажмите кнопку **«Создать команду»**.

После сохранения команда будет добавлена в систему, а пользователь станет её администратором. Также будет сгенерирован код приглашения, который можно использовать для подключения других участников.

## 7. Вступление в команду по коду приглашения

Если команда уже создана другим пользователем, можно присоединиться к ней по коду приглашения:

1. перейдите на страницу вступления в команду;
2. введите полученный код приглашения;
3. нажмите кнопку **«Вступить»**.

Если код введён корректно, система добавит пользователя в состав команды.

## 8. Просмотр информации о команде

Для просмотра состава команды:

1. откройте панель управления;
2. выберите нужную команду из списка;
3. перейдите на страницу команды.

На данной странице отображаются:

- название команды;
- код приглашения;
- список участников;
- роли участников;
- дата вступления;
- кнопки управления участниками при наличии административных прав.

Администратор может обновлять код приглашения, повышать и понижать роли, а также исключать участников. Обычный участник может просматривать данные команды и, при необходимости, покинуть её.

## 9. Создание проекта

Для создания проекта выполните следующие действия:

1. на панели управления нажмите кнопку **«Новый проект»**;
2. выберите команду, в рамках которой создаётся проект;
3. укажите название проекта;
4. при необходимости заполните описание;
5. нажмите кнопку **«Создать проект»**.

После создания проект станет доступен в списке проектов, а пользователь сможет перейти к задачам этого проекта.

## 10. Просмотр списка задач проекта

Для перехода к задачам проекта:

1. откройте панель управления;
2. выберите нужный проект;
3. нажмите кнопку перехода к списку задач.

На странице списка задач пользователь может:

- просматривать все задачи проекта;
- выполнять поиск по названию и описанию;
- фильтровать задачи по статусу;
- фильтровать задачи по приоритету;
- фильтровать задачи по исполнителю;
- быстро изменять статус задачи;
- удалять задачу, если она была создана текущим пользователем.

## 11. Создание задачи

Чтобы добавить новую задачу:

1. откройте страницу проекта;
2. нажмите кнопку **«Новая задача»**;
3. укажите название задачи;
4. заполните описание;
5. выберите статус;
6. задайте приоритет;
7. назначьте исполнителя при необходимости;
8. укажите срок выполнения;
9. добавьте теги через запятую;
10. нажмите кнопку сохранения.

После этого задача будет добавлена в проект и появится в списке задач.

Для перехода к визуальному представлению задач:

1. откройте страницу проекта;
2. нажмите кнопку **«Доска»**.

На доске задачи распределяются по колонкам:

- **«К выполнению»**;
- **«В работе»**;
- **«Выполнено»**.

Канбан-доска позволяет быстро оценивать текущий статус проекта и удобна для визуального контроля хода выполнения задач.

Для перехода в карточку задачи:

1. откройте список задач или канбан-доску;
2. нажмите на название нужной задачи.

- полное название задачи;
- текущий статус;
- приоритет;
- описание;
- исполнителя;
- срок выполнения;
- теги;
- комментарии;
- прикреплённые файлы;
- историю активности.

Для редактирования задачи:

1. нажмите кнопку **«Редактировать»**;
2. измените необходимые параметры;
3. сохраните изменения.

3. введите текст сообщения;
4. нажмите кнопку **«Добавить комментарий»**.

Комментарий будет сохранён и сразу отобразится в общем списке комментариев по задаче.

3. выберите файл на устройстве;
4. нажмите кнопку **«Загрузить»**.

После загрузки файл появится в списке вложений. Пользователь сможет скачать его или удалить, если именно он загружал этот файл.

## 18. Типовой сценарий работы пользователя

Ниже приведён рекомендуемый порядок работы в системе:

1. зарегистрироваться или войти в систему;
2. создать новую команду;
3. пригласить участников по коду;
4. создать проект внутри команды;
5. добавить несколько задач;
6. назначить исполнителей и сроки;
7. отслеживать выполнение задач через список и доску;
8. использовать комментарии и вложения для сопровождения работы;
9. анализировать текущее состояние проекта через раздел аналитики.

## 19. Возможные ограничения

При работе с системой следует учитывать следующие особенности:

- доступ к проектам и задачам зависит от участия пользователя в команде;
- административные действия в команде доступны только пользователям с соответствующей ролью;
- удаление комментариев и вложений возможно только для их автора;
- загрузка файлов ограничивается допустимыми форматами;
- аналитика отображается только при наличии доступных пользователю проектов и задач.

## 20. Заключение

Система Canbas предоставляет пользователю полный набор базовых инструментов для организации командной работы над задачами. Последовательное использование возможностей приложения позволяет создать рабочее пространство команды, распределить обязанности, отслеживать прогресс и получать аналитическую информацию по проектной деятельности.
